## Backend Modules

The backend is built with FastAPI (Python) and relies on a clean, modular service-oriented architecture to evaluate deal health and push actionable decisions organically.

### `data/mock_data.py`
**Purpose:** Simulates a live production database for the MVP. It defines the mock pipeline of prospects, stakeholders, and action recommendations generated by the decision engine.

* `get_current_time()`: Helper function returning a UTC-aware datetime, used to generate dynamic "last activity" timestamps ensuring time-based intelligence triggers organically.
* `deals`: A list of dictionaries representing B2B deals holding core fields:
  * `id`, `company`, `stage`, `status`
  * `last_activity`: ISO formatted timestamp.
  * `value`: Estimated deal financial value.
  * `activation_step`: Current step exclusively tracked once the deal transitions to "Signed".
  * `stakeholders`: List containing specific points of contact mapped to role, response boolean logic, and engagement statistics.

### `main.py`
**Purpose:** The minimal access controller and entry file housing standard server initializations natively limiting explicit logic bloat.

* **Setup & Middleware:** Initializes the core `FastAPI()` app and establishes the permissive `CORSMiddleware` hook to align with the React UI.
* **Router Inclusion:** Registers the completely decoupled router from `routes/deals.py`.

### `services/evaluation.py`
**Purpose:** Organizes the primary evaluation methodology tying together core status handling parameters across the system natively syncing outputs cleanly across the backend safely.

* `evaluate_deal(deal: dict)`: Maps standard activity time tracking logic, and securely invokes specific logic functions from `activation.py` and `actions.py` directly generating localized recommendations rendering risk variables and final `action_confidence` values updating the data node dictionary natively.

### `services/activation.py`
**Purpose:** Post-signing execution logic isolated out to track complex milestone checkpoints uniquely validating live-deployment integration limits properly mapping dependencies.

* `evaluate_activation_logic(...)`: Explicitly traces specific role interactions aligned with custom milestones (API Shared, Sandbox, Testing). Processes conditions mapping blockages (e.g. CTO blocking Sandbox) and isolates missing timeline momentum creating localized flags properly identifying `activation_bottleneck` statuses.

### `services/actions.py`
**Purpose:** Standalone artificial candidate intelligence module projecting predictive scoring calculations identifying next-best paths dynamically.

* `generate_and_score_actions(...)`: Compiles a nested mapping algorithm weighing financial numeric bounds safely alongside derived staging limitations and problem flags. Assigns scoring limits evaluating conditions generating multiple action strings, and cleanly selects optimal candidates safely outputting pure metrics natively.

### `services/priority.py`
**Purpose:** Extracts independent pipeline sequencing rendering explicit hierarchy mappings automatically limiting visual noise optimally structuring top priority objects clearly directly mapping UX impacts.

* `prioritize_deals(deals_list: list)`: Safely aggregates string and float maps pushing constraints limits cleanly scaling absolute properties alongside activation checkpoints limits outputting assigned `priority_score` values returning a strictly backward-indexed sorted array.

### `routes/deals.py`
**Purpose:** Dedicated controller module storing HTTP method routes natively organizing requests decoupling presentation handling paths purely targeting data endpoints natively efficiently.

* `GET /api/deals`: Interfaces cleanly with isolated services safely formatting dictionaries through `evaluate_deal` handlers securely returning prioritized schemas purely formatted natively inside arrays dynamically.
* `POST /api/deals/{deal_id}/action`: Safely isolates simulation behaviors directly overriding fallback locks directly modifying dataset states without crossing scope architectures limiting interference directly cleanly rendering UX mitigations organically.
